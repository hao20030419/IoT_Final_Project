# 🚀 快速入門 - 影片壓縮功能

## 5分鐘快速上手

### 步驟1️⃣：啟動應用程式

```bash
python main_gui.py
```

### 步驟2️⃣：上傳影片

```
點擊：[Upload Video] 按鈕
選擇：你的影片檔案 (.mp4, .avi, .mov, .mkv)
```

### 步驟3️⃣：分析影片

```
點擊：[Process Video] 按鈕
等待：系統分析完成（進度條會顯示進度）
查看：右側會顯示效能指標和檢測結果
```

### 步驟4️⃣：壓縮並下載

```
點擊：[Compress & Download] 按鈕
選擇：儲存位置和檔名
完成！壓縮後的影片已保存 ✅
```

### 步驟5️⃣：查看統計資訊

```
查看"影片壓縮資訊"面板：
✓ 原始影格數
✓ 保留影格數  
✓ 壓縮比例
✓ 檔案大小減少率
```

## 核心概念（簡版）

### 什麼被保留？

✅ **YOLO偵測到物體的影格** - 系統檢測到有物體時保留
✅ **關鍵幀** - 每秒至少1幀（即使沒有物體）

### 什麼被丟棄？

❌ **無動作的幀** - 沒有物體、沒有運動的幀會被丟棄

### 結果？

- 📉 幀數減少 80-97%
- 💾 檔案縮小 70-95%
- ⚡ 快速、易於傳輸和儲存

## 常見問題速查

### Q: 為什麼壓縮後影片那麼短？
**A**: 這是對的！系統只保留有物體的幀 + 每秒1幀。如果你的影片大多沒有物體，會很短。

### Q: 壓縮後影片品質會下降嗎？
**A**: 不會。每個保留的幀都是原始品質，只是幀數少了，看起來會"跳躍"。

### Q: 可以改變每秒保留多少幀嗎？
**A**: 可以！編輯 `video_processor.py` 的 `frame_interval` 參數。

### Q: 壓縮需要多久？
**A**: 約每秒處理 10-15 幀（CPU）。例如10分鐘的影片約需2-3分鐘。

## 進階用法

### 調整幀差異敏感度

編輯 `config.py`：
```python
FRAME_DIFF_CONFIG = {
    'threshold': 7000  # 降低 = 更敏感
}
```

### 改變每秒保留的幀數

編輯 `video_processor.py`：
```python
frame_interval = max(1, int(fps) // 2)  # 改為每秒2幀
```

### 修改YOLO模型

編輯 `main_gui.py` 初始化：
```python
self.video_processor = VideoProcessor(yolo_model_size='m')  # 改用medium模型
```

## 故障排除

| 問題 | 解決方案 |
|-----|---------|
| [Compress & Download] 按鈕是灰色 | 先點 [Process Video] 完成分析 |
| 無法選擇儲存位置 | 確保有寫入權限或選擇其他資料夾 |
| 壓縮很慢 | 這是正常的，CPU處理需要時間 |
| 檔案很小沒有物體 | 預期行為！沒有物體所以保留很少幀 |
